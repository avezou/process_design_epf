Main Description

The goal of this activity is to build the rule set at a level useful to validate the integration with the core business application. The process is simple and includes recurring tasks as outlined by the following process map



The tasks are

  • Defining what is a rule set
  • Put in place an continuous integrated build process which can extract a rule set and execute the suite of tests on it automatically.
  • Develop unit tests
  • Develop rules
  • Execute rules
  • Enhance the object models
  • Log issues
  • Persist the repository in change control
