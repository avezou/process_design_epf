Task: Build Test Scenarios
Work with analysis team to define concrete scenario with real data so that we can later on test the process and the rules.
Disciplines: Rule Analysis

Main Description

Developing without testing has no sense today (we hope!). Developing rules deployed in rule engine helps us supporting efficiently a Test Driven Development approach. Rule set can be isolated early in the development process and can be tested in a sandbox environment. Writing tests before the rule makes testing part of a validation feedback loop.  So during the harvesting phase the analysis team needs to develop test scenario and data elements to support the rule writing and testing. Working on concrete scenario leads to clarify ambiguities, find holes in the decision processing, and enhance rules decision coverage, and the overall quality.

At this level the scenario description can be built as user story with persona involvement, and data point to illustrate the scenario.
